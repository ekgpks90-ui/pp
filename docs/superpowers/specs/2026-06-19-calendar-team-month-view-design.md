# Calendar 전체보기(대표) — 팀 사이드바 + 월 달력 리뉴얼

작성일: 2026-06-19
브랜치: `feat/calendar-react` (워크트리: `vibe-projects-calendar`)
대상: 대표(owner) 전용 Calendar 전체보기. 직원·팀장 화면은 변경 없음.

## 배경 / 목적

현재 대표 Calendar의 "전체 보기"는 프로젝트마다 한 줄짜리 가로 간트 막대를 쌓아
보여준다. 프로젝트 수가 많고 팀 구분이 없어, 어느 팀이 무엇을 언제 하는지 한눈에
파악하기 어렵다.

목표: **맨 왼쪽에 팀을 두고, 팀을 고르면 그 팀 프로젝트들이 월 달력 위에 기간 막대로
보이도록** 바꿔 기간 가독성과 팀별 현황 파악을 높인다.

## 레이아웃 (3단)

```
┌── 팀 사이드바 ──┐┌── 본문: 월 달력 (프로젝트행 + 가로 날짜) ──────────┐
│ ▸ 전체     (11) ││ 프로젝트        1  5  10  15  20  25  30           │
│   UI/UX     (6) ││ ─────────────  ──────────────────────────────────  │
│   브랜드     (2) ││ 모아커머스 앱    ████                               │
│   콘텐츠     (2) ││ 테크스타트 UI    ▒▒▒▒▒▒▒                            │
│   영상·모션  (1) ││ 핏라이프 앱           ▒▒▒▒▒                         │
│   기타      (1) ││ 블루밍헬스          ▒▒▒▒▒▒                          │
└─────────────────┘└────────────────────────────────────────────────────┘
```

## 1. 팀 분류 — `processId` 기반 런타임 매핑

`workItem`에 `team` 필드를 추가하지 않고, 기존 `processId`로 팀을 도출한다
(데이터 중복·동기화 부담 제거).

| processId | 카테고리 | 팀 이름 |
|---|---|---|
| `pc-1` | UI/UX 디자인 | UI/UX 디자인팀 |
| `pc-2` | 브랜드 & 인쇄물 | 브랜드 디자인팀 |
| `pc-3` | 디지털 콘텐츠 | 콘텐츠 디자인팀 |
| `pc-4` | 영상 & 모션 | 영상·모션팀 |
| (없음) | — | 기타 |

- `helpers.js`에 상수 `PROCESS_TEAM`과 함수 `getProjectTeam(workItem)` 추가.
- `processId`가 없는 캘린더 표시 프로젝트(예: wi-8 디자인 시스템 컴포넌트 정리)는
  **"기타"** 팀으로 묶어 누락 없이 노출.

### 현재 데이터 기준 분포 (참고)
- UI/UX 디자인팀(pc-1): wi-5, wi-7, wi-9, wi-11, wi-13, wi-15 — 6
- 브랜드 디자인팀(pc-2): wi-10, wi-17 — 2
- 콘텐츠 디자인팀(pc-3): wi-6, wi-12 — 2
- 영상·모션팀(pc-4): wi-16 — 1
- 기타(없음): wi-8 — 1
- (회의·연차·반복 업무는 기존 규칙대로 캘린더에서 제외)

## 2. 사이드바 — 팀 목록 (필터)

- 기존 "프로젝트" 목록 nav를 **팀 목록**으로 교체.
- 항목: `전체` + 4개 팀 + `기타`. 각 항목 우측에 프로젝트 수 뱃지.
- 선택 시 본문 달력을 해당 팀 프로젝트로 필터(`전체`는 전부).
- **owner 전용.** 직원·팀장은 기존 프로젝트별 화면 유지.

## 3. 본문 — 월 달력 (프로젝트행 + 가로 날짜)

- 기존 `AllProjectsTimeline`을 확장하여 사용.
- 좌측: 프로젝트명 칸(+ 상태 뱃지). 우측: 한 달 날짜 격자(주말 음영·오늘 강조) + 기간 막대.
- 막대 색 = 상태(진행 중/지연/시작 전) 기준 (기존 `STATUS_COLOR`/`WORK_ITEM_TYPE_COLOR` 재사용).
- 막대 클릭 → 기존 `CalendarDetailPanel` 유지.
- 프로젝트명 클릭 → 기존 `DetailPanel`(업무 상세) 또는 프로젝트별 모드 진입(아래 4 참조).

## 4. 프로젝트별 모드 진입

- 상단 "전체보기 / 프로젝트별" 토글은 **유지**.
- 사이드바는 팀 필터 전용이므로, **프로젝트별 모드 진입은 본문 프로젝트명 클릭**으로 한다.
- 프로젝트별 모드(단계·담당자 간트)는 기존 로직 그대로.

## 유지되는 것 (변경 없음)

- 상단 월 이동(◀ ▶), 전체보기/프로젝트별 토글, 프로젝트별 단계·담당자 간트.
- `CalendarDetailPanel`, `DetailPanel`.
- 직원·팀장 캘린더 화면.
- 제품 규칙: 캘린더에 회의·연차·반복 업무 미표시, 업무항목 완료 상태 없음.

## 영향 범위

- `src/components/CalendarPage.jsx` — 사이드바/본문 구조 변경, 팀 필터 상태 추가.
- `src/data/helpers.js` — `PROCESS_TEAM`, `getProjectTeam` 추가.
- (필요 시) `AllProjectsTimeline`을 별도 파일로 분리 검토.

## 비고

- `team` 필드를 데이터에 직접 넣지 않는 이유: 프로젝트-프로세스가 1:1이라 `processId`로
  충분히 도출 가능하며, 데이터 이중화 시 동기화 버그 위험이 있음.
