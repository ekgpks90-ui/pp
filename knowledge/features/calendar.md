# Calendar

## 목적
진행 중인 프로젝트의 단계별 일정을 Gantt 타임라인으로 조회한다.

---

## 레이아웃

좌우 분리 구조:
- **좌측**: 진행 중 프로젝트 사이드바 (세로 네비게이션)
- **우측**: 선택된 프로젝트의 Gantt 타임라인

---

## 프로젝트 사이드바 (좌측)

- 진행 중인 프로젝트명 목록 표시
- 프로젝트 수가 많아지면 스크롤로 관리
- 프로젝트 클릭 시 우측에 해당 프로젝트의 Gantt 타임라인 렌더링
- 초기 상태: "← 좌측에서 프로젝트를 선택하세요" 안내 표시

---

## Gantt 타임라인 (우측)

### 표시 구조
- 상단: 선택된 프로젝트명 + `< 월 >` 이동 버튼
- 프로세스 단계별로 행을 생성
- 한 단계에 담당자가 2명 이상인 경우 담당자별로 행 분리 (첫 번째 행에 단계명, 이후 행은 공백)

### 각 행 표시 항목
- 단계명 (좌측)
- 담당자 아바타 + 이름 (좌측)
- Gantt 바 (우측): 해당 월 기준 시작일~마감일 위치 계산

### Gantt 바 규칙
- 단계 기간 = 해당 단계 담당자 중 가장 빠른 날짜 ~ 가장 늦은 날짜
- 담당자가 수락 시 선택한 날짜 기준으로 자동 반영
- 담당자별 컬러 구분
- 해당 월에 기간이 걸쳐 있는 경우 부분 표시

---

## 예외 프로세스 표시

- 저장된 프로세스 템플릿 외 수동 등록된 예외 프로세스는 별도 컬러로 구분
- 기존 프로세스: 기본 컬러 / 예외 프로세스: 구분 컬러 (예: 주황)

---

## 필터

- 팀: 전체 프로젝트 일정
- 나: 내가 참여 중인 프로젝트 일정

---

## 조회 권한

| 역할 | 조회 범위 |
|------|-----------|
| Owner | 전체 프로젝트 |
| Manager | 팀 프로젝트 |
| Member | 팀 프로젝트 (조회만) |

---

## 캘린더에 표시하지 않는 항목

- 연차
- 작업세션
- 업무요청
- 단순 업무항목 (프로세스 기반이 아닌 것)
