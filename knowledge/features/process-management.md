# 프로세스 관리 (Process Management)

## 목적
반복 사용되는 업무 프로세스 템플릿을 등록·관리하여 업무 요청 시 자동 매핑한다.

## 권한
Manager / Owner

## 위치
Team Status 탭 내 별도 섹션 또는 독립 페이지

---

## 프로세스 템플릿 구조

| 필드 | 설명 |
|------|------|
| 프로세스명 | 예: 배너관리 프로세스, 콘텐츠 제작 프로세스 |
| 키워드 목록 | 프로젝트명 자동 매핑에 사용 (예: 배너, 배너작업, banner) |
| 단계 목록 | 순서 있음. 각 단계: 단계명 + 기본 담당 역할(선택) |

---

## 템플릿 관리 기능

- 템플릿 생성 / 수정 / 비활성화 (물리삭제 금지)
- 단계 순서 변경 (드래그 또는 위/아래 버튼)
- 키워드 추가 / 삭제

### 생성 시 필수 입력값
- 프로세스명
- 트리거 키워드 (1개 이상)
- 프로세스 단계 (1개 이상, 단계명 입력)

### 수정 가능 항목
- 프로세스명
- 트리거 키워드 (추가/삭제)
- 단계명
- 단계 순서
- 단계 추가 / 삭제

---

## 키워드 매칭 규칙

- 프로젝트명에 키워드가 **포함**되면 매칭 (완전 일치 불필요)
- 대소문자 구분하지 않음
- 예: 키워드 "배너" → 프로젝트명 "하반기 배너작업" → 매칭됨
- 복수 프로세스 매칭 시 팀장이 직접 선택

---

## 업무 요청 연동 흐름

1. 팀장이 업무 요청 생성 시 프로젝트명 입력
2. 입력값에 등록된 키워드가 포함되면 매칭 프로세스 자동 추천
3. 팀장이 추천 프로세스 선택 → 단계별 업무 목록 + 담당자 배치 UI 표시
4. 단계별 담당자 지정 후 일괄 업무요청 전송
5. 각 단계는 독립 WorkRequest로 생성됨

### Case 1 — 매칭 프로세스 있음
1. 프로젝트명 입력 → 키워드 기반 매칭 프로세스 자동 표시
2. 매칭 프로세스가 2개 이상이면 팀장이 직접 선택
3. 프로세스 선택 → 단계 목록 + 단계별 담당자 배치 UI 노출
4. 담당자 배치 완료 후 일괄 전송

### Case 2 — 매칭 프로세스 없음
1. 프로젝트명 입력 → "등록된 프로세스 없음" 안내 표시
2. 모달 내 인라인으로 프로세스 즉시 등록 (프로세스 관리 페이지 이동 없음)
3. 등록 완료 후 단계별 담당자 배치 UI 노출
4. 담당자 배치 완료 후 일괄 전송
