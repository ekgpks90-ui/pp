# 프로세스 관리 (Process Management)

## 목적
반복 사용되는 업무 프로세스 템플릿을 등록·관리한다.
팀원 업무 현황에서 업무 배정 시 저장된 프로세스를 선택하여 단계별 인원을 배정한다.

## 권한
Manager / Owner

## 위치
Team Status 탭 내 별도 섹션 또는 독립 페이지

---

## 프로세스 템플릿 구조

| 필드 | 설명 |
|------|------|
| 프로세스명 | 예: 배너관리 프로세스, 콘텐츠 제작 프로세스 |
| 단계 목록 | 순서 있음. 각 단계: 단계명 |

---

## 템플릿 관리 기능

- 템플릿 생성 / 수정 / 비활성화 (물리삭제 금지)
- 단계 순서 변경 (드래그 또는 위/아래 버튼)

### 생성 시 필수 입력값
- 프로세스명
- 프로세스 단계 (1개 이상, 단계명 입력)

### 수정 가능 항목
- 프로세스명
- 단계명
- 단계 순서
- 단계 추가 / 삭제

---

## 업무 요청 연동 흐름

1. 팀장이 팀원 업무 현황에서 업무 배정 시 저장된 프로세스 목록에서 직접 선택
2. 프로세스 선택 → 단계 목록 + 단계별 담당자 배치 UI 표시
3. 단계별 담당자 지정 후 일괄 업무요청 전송
4. 각 단계는 독립 WorkRequest로 생성됨
5. 수신 측(직원/팀장) 홈 업무요청 위젯에 프로젝트명 + 프로세스명이 함께 표시됨
