# 업무요청 (Work Request)

## 목적
팀장이 팀원에게 업무를 요청하는 기능이다.
저장된 프로세스 템플릿을 선택하여 단계별 담당자를 배정할 수 있다.

## 생성 권한
Manager / Owner

---

## 업무요청 생성

### 필수 입력값
- 프로젝트명
- 시작일
- 마감일
- 전달 팀
- 중요도 (`일반` / `긴급`)

### 프로세스 연동

1. 저장된 프로세스 목록에서 직접 선택
2. 프로세스 선택 → 단계 목록 + 단계별 담당자 배치 UI 노출
3. 단계별 담당자 지정 후 일괄 전송
4. 각 단계는 독립 WorkRequest로 생성됨

> 프로세스는 선택 사항이며, 프로세스 없이 단순 업무요청도 생성 가능하다.

---

## 업무요청 상태

| 상태 | 설명 |
|------|------|
| 미배정 | 전달 팀에 도착했지만 담당자가 배정되지 않은 상태 |
| 수락 대기 | 담당자가 배정되어 응답을 기다리는 상태 |
| 수락 | 업무요청 수락 완료 상태 |
| 거절 | 업무요청 거절 완료 상태 |

---

## 수락 / 거절

### 수락 (Member 기준)
- 수락 시 업무항목이 Weekly Tasks에 자동 추가됨
  - 프로젝트명·시작일·마감일 그대로 자동 추가 (팀장 설정값 그대로)
- Daily Todo: 수락 시 직원이 날짜 선택 → 선택한 날짜에 세부업무항목 자동 생성
- 팀장은 프로젝트 시작일·마감일만 설정, 단계별 날짜는 직원이 수락 시 직접 선택

### 거절
- 거절 시 사유 입력 필수
- 거절 사유는 고정 목록 사용 (MVP)

---

## 홈 업무요청 위젯 표시

- 프로젝트명 + 프로세스명이 함께 표시됨

---

## 업무 재배정

- 권한: Manager / Owner
- Member는 재배정 불가
