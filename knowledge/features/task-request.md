# 업무요청 (Work Request)

## 목적
팀장이 팀원에게 업무를 요청하는 기능이다.
프로젝트명을 입력하면 등록된 프로세스 템플릿과 키워드 매칭이 실행되어 단계별 담당자를 배정할 수 있다.

## 생성 권한
Manager / Owner

---

## 업무요청 생성

### 필수 입력값
- 프로젝트명
- 시작일
- 마감일
- 전달 팀
- 중요도 (`일반` / `긴급`)

### 프로세스 연동

프로젝트명 입력 시 키워드 매칭이 실행된다.

**Case 1 — 매칭 프로세스 있음**
1. 프로젝트명 입력 → 키워드 기반 매칭 프로세스 자동 표시
2. 매칭 프로세스가 2개 이상이면 팀장이 직접 선택
3. 프로세스 선택 → 단계 목록 + 단계별 담당자 배치 UI 노출
4. 담당자 배치 완료 후 일괄 전송
5. 각 단계는 독립 WorkRequest로 생성됨

**Case 2 — 매칭 프로세스 없음**
1. 프로젝트명 입력 → "등록된 프로세스 없음" 안내 표시
2. 모달 내 인라인으로 프로세스 즉시 등록 (프로세스 관리 페이지 이동 없음)
3. 등록 완료 후 단계별 담당자 배치 UI 노출
4. 담당자 배치 완료 후 일괄 전송

> 프로세스는 선택 사항이며, 프로세스 없이 단순 업무요청도 생성 가능하다.

### 키워드 매칭 예시
- 프로세스 관리 페이지에 "배너관리" 프로세스가 등록되어 있고 트리거 키워드가 "배너"인 경우
- 프로젝트명을 "하반기 배너작업"으로 입력하면 "배너관리" 프로세스가 자동 제안됨

## 업무요청 상태

| 상태 | 설명 |
|------|------|
| 미배정 | 전달 팀에 도착했지만 담당자가 배정되지 않은 상태 |
| 수락 대기 | 담당자가 배정되어 응답을 기다리는 상태 |
| 수락 | 업무요청 수락 완료 상태 |
| 거절 | 업무요청 거절 완료 상태 |

---

## 수락 / 거절

### 수락 (Member 기준)
- 수락 시 업무항목이 Weekly Tasks에 자동 추가됨
  - 프로젝트명·시작일·마감일 그대로 자동 추가 (팀장 설정값 그대로)
- Daily Todo: 수락 시 직원이 날짜 선택 → 선택한 날짜에 세부업무항목 자동 생성
- 팀장은 프로젝트 시작일·마감일만 설정, 단계별 날짜는 직원이 수락 시 직접 선택

### 거절
- 거절 시 사유 입력 필수
- 거절 사유는 고정 목록 사용 (MVP)
