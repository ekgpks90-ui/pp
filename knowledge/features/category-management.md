7. Category Management
목적
작업세션에서 사용하는 업무카테고리를 관리한다.
업무 데이터 분석의 기준이 되는 분류 체계를 제공한다.

7-1. 업무카테고리 목록 조회
목적
등록된 업무카테고리를 조회한다.

조회 권한
Owner
Manager
Member

표시 항목
	•	카테고리명
	•	상태
	•	생성일
	•	최종 수정일

상태
	•	사용중
	•	비활성화

7-2. 업무카테고리 생성
목적
신규 업무카테고리를 생성한다.

생성 권한
Owner

필수 입력값
	•	카테고리명

생성 완료
업무카테고리 목록에 추가된다.

7-3. 업무카테고리 수정
목적
기존 업무카테고리를 수정한다.

수정 권한
Owner

수정 가능 항목
	•	카테고리명

수정 이력
모든 수정 이력은 데이터베이스에 저장한다.

7-4. 업무카테고리 비활성화
목적
사용하지 않는 업무카테고리를 비활성화한다.

권한
Owner

비활성화 처리
기존 작업세션 데이터는 유지한다.

신규 작업세션
비활성화된 카테고리는 선택할 수 없다.

7-5. 적용 정책
적용 시점
카테고리 변경 사항은 매월 1일부터 적용한다.
생성, 수정, 비활성화 요청은 즉시 사용 상태에 반영하지 않고 예약 변경으로 저장한다.
예약 변경에는 effective_from 값을 저장한다.
effective_from은 다음 월 1일로 설정한다.
매월 1일에 예약 변경을 실제 카테고리 상태에 반영한다.

삭제 정책
업무카테고리는 삭제하지 않는다.
비활성화만 가능하다.

데이터 보존
기존 작업세션의 카테고리 정보는 변경하지 않는다.
