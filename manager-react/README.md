# WorkFlow Manager (React)

바닐라 JS로 만든 원본 `manager` 앱을 **React로 다시 구현한 버전**입니다.
디자인 에이전시의 팀 업무를 관리하는 웹앱으로, 직원·팀장·대표 세 역할을
하나의 앱에서 권한별로 분기해 보여줍니다.

> 제품 기획 기준 문서는 상위 폴더(`vibe-projects`)의 `context/`, `rules/`,
> `knowledge/`를 따릅니다. 용어와 권한 정책은 그쪽이 원본입니다.

## 기술 스택

- **React 19** + **Vite** (빌드/개발 서버)
- **Tailwind CSS v4** (`@tailwindcss/vite` 플러그인)
- 상태 관리는 별도 라이브러리 없이 `App.jsx`의 `useState`로 관리

## 실행 방법

```bash
npm install      # 의존성 설치 (최초 1회)
npm run dev      # 개발 서버 실행 (http://localhost:5173)
npm run build    # 프로덕션 빌드 (dist/ 생성)
npm run preview  # 빌드 결과 미리보기
npm run lint     # ESLint 검사
```

## 역할(Role) 기반 구조

이 앱의 핵심은 **페이지를 역할별로 복제하지 않는다**는 점입니다.
원본의 직원(app)/대표(ceo)/팀장(manager) 3벌을 하나로 통합하고,
"역할마다 다른 부분"은 전부 `src/data/roles.js`의 권한 함수로 분기합니다.

| 역할 | 값 | 설명 |
|------|------|------|
| 직원 | `member` | Process(프로세스 템플릿) 페이지 미노출 |
| 팀장 | `manager` | 전체 페이지 노출, 팀 단위 관리 권한 |
| 대표 | `owner` | 전체 페이지 + 전체 조회 권한 |

역할은 상단바(Topbar)의 역할 스위처로 전환하며 화면을 확인할 수 있습니다.
권한 판단은 `canViewPage`, `canApproveLeave`, `canCreateWorkRequest` 등
`roles.js`의 함수로만 처리합니다.

## 페이지 구성

| 페이지 | 컴포넌트 | 설명 |
|--------|----------|------|
| 홈 | `HomePage` / `CeoDashboard`(대표) | 직원·팀장은 주간 업무항목·작업세션·KPI·업무요청. 업무 요청 **수락**(`AcceptModal`) 시 단계별 데드라인이 함께 표시되고, 수락하면 단계가 **오늘 할 일** 작업세션으로 자동 추가되며 단계명 옆에 `~M/D` 데드라인이 회색으로 노출됨. 데드라인은 단일 기준으로 파생: **단계 데드라인(`stepDeadlines[stepId]`) → 세션 데드라인(`session.deadline`) → 업무 마감일(`end`)**, 없으면 "마감 없음"(고정 업무 등). 표기는 `fmtDeadline`. 이번 주 업무 **상세보기**(`DetailPanel`)의 참여자 프로세스 단계에도 단계 데드라인(`item.stepDeadlines`)이 함께 표시됨. 대표는 경영 대시보드: 상단 **기간 전환**(주간/월간/분기) + **KPI 스트립 5지표**(평균 가동률·과부하/여유·진행/지연·예상 대비 실제 오차·이번 달 총 투입) + 3열 — **프로젝트 투자 현황**(탭 전체·진행 중·지연·이번 주 납기 + 진행률·예상→실제 투입(오차율)·인원/공수·상태, 금액 ON/OFF 토글, 행 클릭 시 상세 슬라이드) · **업무 집중도 추이(예시)+결재함**(계약·예산·착수/종료 결재 안건 승인/반려/상세) · **AI 추천 액션**(지연·과부하·여유·납기 신호별 문제+추천+버튼) |
| 캘린더 | `CalendarPage` | 직원·팀장은 **프로젝트별**(단계·담당자 타임라인). **대표는 3단 구조** — [팀 사이드바] → [선택 팀 프로젝트 리스트] → [월 캘린더(팀 색상 멀티데이 바)]. 팀 클릭 시 리스트·캘린더가 해당 팀으로 필터, 프로젝트 클릭 시 우측에 **홈식 프로젝트 상세**(`CeoProjectDetail` — 투입 원가·공수·진행률·단계, '업무 수정'으로 편집 폼 전환). 회의·연차 제외. 직원은 조회 전용(`canEditCalendar`), 팀장·대표는 수정 가능 |
| 팀원 현황 | `TeamStatusPage` | 직원·팀장은 팀원별 주간 업무·담당자 배정. **업무요청** 모달(`NewRequestModal`)의 **요청 팀 드롭다운**으로 내 팀·다른 팀·**대표님** 선택 — 팀 선택 시 업무요청(`assignmentRequests`), **대표님 선택 시 결재 폼으로 전환**해 대표 결재함(`approvalItems`)에 `대기`로 등록(유형별 상세 필드·첨부파일 모두 선택 입력). 업무요청 시 **프로세스 단계별 데드라인**(`stepDeadlines`)을 달력으로 지정 가능. **대표는 메뉴 없음**(인력 현황은 홈 KPI·리포트 인력 탭에서) |
| 프로세스 | `ProcessPage` | 프로세스 템플릿 관리(추가/수정/삭제). 직원은 메뉴 없음. **대표도 팀장과 동일 화면 사용** |
| 미팅룸 | `MeetingRoomPage` | 회의록 등록·조회, AI 요약(카드 노출), 액션아이템, 팀 탭 + 검색(회의명·참석자·AI요약). 대표는 전체 회의 조회 |
| 연차 | `LeavePage` | 연차 신청·조회·승인 |
| 리포트 센터 | `CeoReportCenter` | **대표 전용 메뉴.** 6개 탭(행/카드 클릭 시 우측 슬라이드): **수익성**(거래처별 **마진**=계약금−원가, 완료 시 예상 마진, 초과 패턴 경고) · **프로젝트**(위험 자동 강조·필터/정렬·예상→실제 오차율, 슬라이드=`CeoProjectDetail`) · **결재**(상태/유형 필터·이력·상세+타임라인+승인/반려) · **인력**(톤 배너·재배분 추천·팀별/개인별 가동률) · **연차**(승인 판단근거·이번 주 현황·잔여 막대) · **Export**(4단계 선택 UI, 다운로드/자동발송=준비 중). 계약금은 `approvalItems`(계약 승인) 기반 실데이터, 매출/수익률만 예시. 완료율·재작업률·야근비율 제외 |
| 마이페이지 | `MyPage`(직원·팀장) / `CeoMyPage`(대표) | 직원·팀장은 작업 히스토리·카테고리별 작업시간·AI 업무패턴 요약·미니 캘린더. **대표는 5섹션 개인 대시보드**(`CeoMyPage`): **프로필**(직책·조직·입사일, 연락처 미등록) · **계정 설정**(알림·권한·프로필·비번 — 백엔드 없어 '준비 중') · **AI 브리핑**(오늘 일정·회의·승인 요청 수 + 추천사항) · **오늘/최근 일정**(회의·세션 시간순) · **미니 캘린더**(회의·일정·연차 점, 날짜 클릭 시 그 날 일정 표시). 연차는 상태 점만(총/사용/잔여 숫자 없음 — 규칙) |

> **대표(owner) 전용 화면**: Home은 화면 성격이 완전히 달라 owner일 때
> 별도 컴포넌트(`CeoDashboard`)로 교체되고, Report Center는 owner에게만 노출됩니다.
> (프로세스는 대표도 팀장과 동일한 `ProcessPage` 사용. 팀원 현황·연차관리 메뉴는 대표에서 제거 —
> 대표 전용 `CeoProcessPage`·`CeoCompanyStatus`는 폐기) 데이터는 한 벌(`state.js`)을 공유하며, 카드/섹션 UI는 `CeoUI.jsx`에
> 모았습니다. 기획 규칙상 "완료" 상태는 만들지 않으므로 모든 지표는 진행 중·지연으로 표현하고,
> 매출 등 데이터 없는 항목은 "예시"로 표기합니다.
>
> 대표 홈의 **프로젝트 투자(투입 비용)**는 개인 월급이 아니라 **직급별 단가**(`state.js`의
> `gradeRates`, 팀원의 `grade` 필드)로 계산합니다(`helpers.js`의 `projectCost`·`dailyRateSum` 등).
> **대표 결재함**은 신규 데이터 `approvalItems`(계약·예산·프로젝트 착수/종료 4종)를 쓰며, 승인/반려는
> `App.jsx`의 `approveItem`·`rejectItem` 핸들러가 처리합니다. 팀장이 팀원 현황의 업무요청 모달에서
> '대표님'을 골라 보낸 결재는 `App.jsx`의 `addApprovalItem`이 결재함 맨 위에 `대기`로 추가합니다(필드는
> 모두 선택 입력이라 빈 값은 미리보기·상세에서 자동 생략). **연차는 홈에 두지 않고** 리포트 연차
> 탭에서 전담합니다. (상세 기획: `../context/ceo-experience.md`)

## 팀장 모바일 앱 (`?mobile`)

`http://localhost:5173/?mobile` 로 접속하면 데스크탑 대신 **팀장 전용 모바일 앱**이
렌더됩니다(`src/mobile/`). 데이터·파생규칙(`src/data/`)은 데스크탑과 그대로 공유하되,
결재·시간기록 등은 모바일 내부 `useState` 복사본에서 처리해 데스크탑에 영향을 주지 않습니다.

설계 원칙: **모바일 = 파악·결정, 데스크탑 = 편집·관리.** 재배정·프로세스 편집·일정
수정 같은 복잡 편집은 제공하지 않습니다(보기 전용).

| 하단 탭 | 핵심 |
|---------|------|
| 홈 | 확실한 사실 요약 3종(지연/마감 임박/대기 결재) · 연차 배너 · 긴급 업무요청 2건 즉시 수락/거절 · 빠른 메뉴 |
| 팀원 | 긴급→업무량 순 정렬 · 진행률(완료/전체) · 연차 흐림 처리 · 상세는 보기 전용 |
| 프로젝트 | 지연→진행중→시작전 정렬 · 상세는 단계별 세로 리스트(간트 대체, 보기 전용) |
| 결재 | 업무요청/연차 세그먼트 · 처리됨 토글 · 수락(수락만)/거절·반려 사유 필수 · 연차 사실 기반 판단근거 |
| 내 업무 | 오늘 현황 · 이번 주 업무(요일별 접기) · 오늘 할 일 시간기록(시작/종료) · 업무 추가 |

> **표시 원칙 — 확정 사실만:** 마감일·완료 여부·건수처럼 확정 가능한 값만 표시합니다.
> 가동률(주간 업무 수 ÷ 임의 기준)·과부하/여유 분류·추이처럼 기준이 모호한 추정 지표는
> 표시하지 않습니다. 지연(마감 경과)·진행률(완료/전체)은 확정 가능하므로 그대로 사용합니다.

화면 스크린샷은 `screenshots/mobile/` 에 있습니다.

```
src/

├── components/         # 데스크탑 페이지 및 UI 컴포넌트
├── mobile/             # 팀장 모바일 앱 (?mobile) — data/ 공유

│   ├── derive.js       # 모바일 파생 계산 (확정 사실만)
│   ├── theme.js · ui.jsx
│   ├── tabs/           # 홈·팀원·프로젝트·결재·내 업무
│   └── screens/        # 팀원 상세·프로젝트 상세·보기 전용(회의록/캘린더/프로세스)
└── data/
    ├── state.js        # 목 데이터 (사용자·업무항목·회의·연차 등)
    ├── roles.js        # 역할 정의 및 권한 판단 함수
    └── helpers.js      # 날짜·시간 유틸 (ISO 변환, 정렬, 진행률·지연 파생 등)
```

## 용어 (원본 기획 기준)

혼동하기 쉬운 핵심 용어를 정리합니다. 자세한 정의는 상위 폴더
`context/glossary.md`를 따릅니다.

- **업무요청(Work Request)** — 담당자를 배정받는 작업 요청
- **업무항목(Work Item)** — 참여자가 붙는 실제 업무 단위
- **작업세션(Work Session)** — 작성자가 기록하는 개별 작업 (물리 삭제)

업무요청 수락이 곧 업무항목 자동 생성을 의미하지 않으며,
업무항목에는 완료 상태나 타이머 개념이 없습니다.
