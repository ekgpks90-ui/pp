# Category Management Wireframe

## 목적

작업세션 분석용 업무카테고리를 관리한다.

## 기본 레이아웃

```txt
┌────────────────────────────────────────────────────────────────────┐
│ Category Management                         [+ 카테고리 변경 요청] │
├────────────────────────────────────────────────────────────────────┤
│ 현재 카테고리 목록                                                 │
│ ┌────────────┬────────────┬────────────┬────────────┐             │
│ │ 카테고리명 │ 상태       │ 적용 시작일 │ 최종 수정일 │             │
│ ├────────────┼────────────┼────────────┼────────────┤             │
│ │ 기획       │ 사용중     │ 2026-06-01 │ 2026-05-20 │             │
│ │ 디자인     │ 사용중     │ 2026-06-01 │ 2026-05-20 │             │
│ └────────────┴────────────┴────────────┴────────────┘             │
├────────────────────────────────────────────────────────────────────┤
│ 예약 변경 목록                                                     │
│ 변경 유형 / 카테고리명 / 변경 내용 / effective_from / 상태          │
└────────────────────────────────────────────────────────────────────┘
```

## 변경 요청 모달

```txt
┌─────────────────────────────┐
│ 카테고리 변경 요청          │
├─────────────────────────────┤
│ 변경 유형                   │
│ ○ 생성                      │
│ ○ 수정                      │
│ ○ 비활성화                  │
│                              │
│ 카테고리명 [             ]   │
│ 적용 예정일: 다음 월 1일     │
│                              │
│ [취소] [예약 저장]           │
└─────────────────────────────┘
```

## 정책 메모

- Owner만 생성/수정/비활성화할 수 있다.
- Manager/Member는 조회만 가능하다.
- 변경은 즉시 반영하지 않고 예약 변경으로 저장한다.
- `effective_from`은 다음 월 1일이다.
- 카테고리는 삭제할 수 없다.
- 기존 작업세션의 카테고리 정보는 변경하지 않는다.
