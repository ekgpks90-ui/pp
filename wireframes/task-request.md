# Task Request Wireframe

## 목적

팀 간 업무요청을 생성, 배정, 수락, 거절한다.

## Manager / Owner 화면

```txt
┌────────────────────────────────────────────────────────────────────┐
│ Task Request                                      [+ 새 업무요청]  │
├────────────────────────────────────────────────────────────────────┤
│ Filters: [전달 팀] [상태] [중요도] [기간] [검색]                  │
├──────────────────────────────┬─────────────────────────────────────┤
│ 업무요청 목록                 │ 상세 패널                           │
│                              │                                     │
│ [미배정] 랜딩페이지 제작      │ 업무명: 랜딩페이지 제작              │
│ 요청자: 김팀장               │ 요청자 / 요청 팀                    │
│ 전달 팀: 디자인팀            │ 전달 팀: 디자인팀                   │
│ 중요도: 긴급                 │ 담당자: [담당자 선택] [배정]         │
│                              │ 시작일 / 마감일                     │
│ [수락 대기] 쇼핑몰 리뉴얼     │ 상세내용                            │
│ 담당자: 박디자               │                                     │
│                              │ [수정] [이력]                       │
└──────────────────────────────┴─────────────────────────────────────┘
```

## 담당자 화면

```txt
┌────────────────────────────────────────────────────────────────────┐
│ 업무요청 상세                                                      │
├────────────────────────────────────────────────────────────────────┤
│ 업무명                                                            │
│ 요청자 / 요청 팀 / 전달 팀                                         │
│ 시작일 / 마감일 / 중요도                                           │
│ 상세내용                                                          │
│                                                                  │
│ [수락] [거절]                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## 거절 모달

```txt
┌─────────────────────────────┐
│ 업무요청 거절               │
├─────────────────────────────┤
│ 거절 사유                   │
│ ○ 일정 불가                 │
│ ○ 담당 범위 아님            │
│ ○ 리소스 부족               │
│ ○ 요청 내용 불명확          │
│ ○ 기타                      │
│                             │
│ 기타 상세 사유              │
│ [                         ] │
│                             │
│ [취소] [거절 완료]          │
└─────────────────────────────┘
```

## 완료 기준 메모

- 업무요청은 전달 팀으로 먼저 수신된다.
- 담당자 배정 전에는 미배정 업무로 표시한다.
- 전달 팀의 Manager가 담당자를 배정한다.
- 담당자만 수락/거절할 수 있다.
- 수락해도 업무항목은 자동 생성되지 않는다.
- 수락된 업무요청은 Weekly Tasks 추가 액션으로 이어진다.
