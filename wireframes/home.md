## Figma 참고 레이아웃 반영

참고 Figma 노드:

- `2Hb9msCj0pKKYuuxvqBxe3 / 184:21003`

참고할 점:

- 좌측 고정 사이드바
- 상단 주차 이동 컨트롤
- 상단 검색, 알림, 프로필
- 본문 3컬럼 구성
  - 왼쪽: Weekly Tasks
  - 가운데: Today/Daily Todo
  - 오른쪽: KPI와 보조 액션 패널
- 알림 패널은 상단 알림 버튼에서 드롭다운으로 노출
- 업무요청 카드는 별도 팝오버 또는 우측 패널로 노출 가능
- 업무항목 추가/수정은 우측 슬라이드 패널 또는 모달로 처리
- 삭제 확인은 작은 확인 모달로 처리
- Weekly Tasks 업무항목 버튼을 누르면 월~금 계획 입력 묶음이 있는 작성 패널이 열린다
- Daily Todo는 Weekly Tasks에서 업무항목을 선택했을 때 작업세션 작성 영역이 열린다

반영하지 않을 점:

- 타이머/녹화/Current Recording UI는 MVP 정책상 제외한다.
- Today Schedule이라는 명칭은 Daily Todo 정책과 충돌할 수 있으므로 `Daily Todo` 또는 `오늘 작업세션`으로 사용한다.
- 기존 Figma의 항목명과 카피는 현재 기획 문서 기준으로 교체한다.

## UX 메모

- KPI는 숫자 중심으로 작고 밀도 있게 배치한다.
- Figma 참고안처럼 Weekly Tasks, Daily Todo, Side Panel을 3컬럼으로 배치한다.
- 업무항목을 선택하면 Daily Todo가 해당 업무항목의 작업세션으로 전환된다.
- 작업세션 시간 입력은 더블클릭으로 열리는 인라인 입력을 우선한다.
- 작업세션 행은 체크박스를 맨 앞에 두고, 삭제 버튼은 hover 시 노출한다.
- 업무항목 추가 패널은 월~금 계획 입력 묶음으로 구성한다.
- 추가 버튼으로 묶음을 하나 더 만들 수 있다.
- 작업세션이 없는 경우 빈 상태를 표시하고 `업무세션에 포함` 또는 `작업세션 추가` 액션을 제공한다.
- Weekly Tasks에서 컬러는 개별 업무항목 카드가 아니라 오늘 요일 섹션에만 표시한다.
- 오늘이 월요일이면 월요일만 컬러 섹션으로 표시하고, 다른 요일은 그레이 섹션으로 표시한다.
- 월요일 고정업무 예시는 `주간회의`이다.
- 고정업무 아래에만 구분선을 표시한다.
- 업무항목은 요일 안에서 고정 업무, 긴급 업무, 일반 업무 순서로 정렬한다.
- 업무항목 행은 리스트 형태로 표시하고, 업무유형은 고정 마커/긴급 점/일반 점으로 구분한다.
