WorkFlow Hub - 용어 정의서
목적
본 문서는 서비스 내에서 사용하는 용어를 통일하기 위한 문서이다.
기획자, 디자이너, 개발자, AI(ChatGPT, Claude, Cursor, Codex)는 본 문서를 기준으로 동일한 의미로 용어를 사용한다.

사용자 / 권한
대표 (Owner)
정의
조직 전체를 관리하는 최고 권한 사용자
혼동 주의
	•	관리자라는 용어 대신 대표를 사용한다.

팀장 (Manager)
정의
팀 단위 업무를 관리하는 사용자
혼동 주의
	•	팀장도 업무 수행자이다.
	•	대표와 동일한 권한을 가지지 않는다.

직원 (Employee)
정의
실제 업무를 수행하는 사용자
혼동 주의
	•	업무요청 생성 권한은 없다.
	•	업무항목 참여자 변경 권한은 없다.

조직
회사 (Company)
정의
서비스를 사용하는 조직

팀 (Team)
정의
회사 내 업무 조직 단위
예시
	•	디자인팀
	•	개발팀
	•	마케팅팀

업무
업무항목 (Work Item)
정의
프로젝트 역할을 수행하는 업무 단위
서비스 내 별도의 프로젝트 엔티티는 존재하지 않으며 업무항목이 프로젝트 역할을 수행한다.
예시
	•	쇼핑몰 리뉴얼
	•	랜딩페이지 제작
	•	브랜드 리뉴얼
	•	디자인 시스템 구축
혼동 주의
	•	프로젝트라는 용어는 업무항목과 동일한 의미로 사용한다.
	•	별도의 프로젝트 엔티티는 존재하지 않는다.
	•	업무항목은 여러 명의 참여자를 가질 수 있다.
	•	업무항목에는 완료 상태가 존재하지 않는다.

업무유형 (Work Type)
정의
업무항목의 성격을 구분하는 값
유형
	•	일반
	•	고정
	•	긴급

업무카테고리 (Work Category)
정의
작업세션을 분류하기 위한 분석용 카테고리
예시
	•	기획
	•	디자인
	•	회의
	•	운영
	•	기타
혼동 주의
	•	업무항목 분류가 아니다.
	•	작업세션 분류이다.

작업세션 (Work Session)
정의
실제 작업 기록 단위
예시
09:00 ~ 11:00
메인 시안 제작
혼동 주의
	•	작업세션은 업무항목에 속한다.
	•	작업세션은 완료 체크가 가능하다.
	•	작업세션은 시간 분석의 최소 단위이다.
	•	작업로그라는 용어는 사용하지 않는다.

업무 계획 / 수행
Weekly Tasks
정의
주간 업무항목을 계획하고 관리하는 화면
혼동 주의
	•	Weekly Tasks는 화면명이다.
	•	업무항목(Work Item)을 관리하는 영역이다.
	•	실제 작업 기록 공간이 아니다.

Daily Todo
정의
일일 작업세션을 관리하는 화면
혼동 주의
	•	Daily Todo는 화면명이다.
	•	작업세션(Work Session)을 관리하는 영역이다.
	•	업무 계획 영역이 아니다.
	•	실제 수행한 작업을 기록하는 영역이다.

업무 요청
업무요청 (Work Request)
정의
특정 사용자에게 업무를 요청하는 기능
출처
	•	클라이언트
	•	내부
	•	회의
	•	기타
혼동 주의
	•	업무항목이 아니다.
	•	수락 시 업무항목으로 전환된다.

요청자 (Requester)
정의
업무요청을 생성한 사용자

담당자 (Assignee)
정의
업무요청을 수신하고 응답하는 사용자
혼동 주의
	•	업무요청에서만 사용하는 용어이다.
	•	업무항목에서는 사용하지 않는다.
	•	업무요청은 먼저 전달 팀으로 수신된다.
	•	전달 팀의 Manager가 담당자를 배정한 후 담당자가 수락 또는 거절한다.

응답기한
정의
업무요청에 응답해야 하는 마감일

신규 요청
정의
생성되었지만 아직 응답되지 않은 업무요청 상태

수락 대기
정의
담당자의 응답을 기다리고 있는 상태

미배정 업무
정의
담당자가 지정되지 않은 업무
혼동 주의
	•	전달 팀에는 도착했지만 아직 담당자가 배정되지 않은 업무요청이다.
	•	Manager가 담당자를 배정해야 수락 또는 거절 단계로 넘어간다.

업무 수행자
참여자 (Participant)
정의
업무항목에 참여하는 사용자
혼동 주의
	•	한 업무항목에는 여러 명의 참여자가 존재할 수 있다.
	•	업무요청 담당자와 다른 개념이다.

작성자 (Author)
정의
작업세션을 생성한 사용자
혼동 주의
	•	작업세션은 작성자 1명만 가질 수 있다.
	•	업무항목 참여자와 다른 개념이다.

재배정
정의
업무항목의 참여자를 변경하는 행위
혼동 주의
	•	업무요청의 담당자 배정과 구분한다.
	•	업무항목에서는 담당자라는 용어를 사용하지 않고 참여자를 사용한다.

업무 상태
진행중 (In Progress)
정의
정상적으로 수행되고 있는 업무 상태

지연중 (Delayed)
정의
마감일이 지났고 미완료 작업세션이 존재하는 상태
혼동 주의
	•	사용자가 직접 선택하지 않는다.
	•	시스템이 자동 계산한다.

캘린더
Calendar
정의
업무 일정과 회의 일정을 조회하는 화면
혼동 주의
	•	업무를 생성하는 화면이 아니다.
	•	등록된 업무항목과 회의를 일정 형태로 조회한다.

팀 캘린더
정의
팀 전체 일정 조회 화면

개인 캘린더
정의
본인 일정만 조회하는 화면

회의
Meeting Room
정의
회의를 생성하고 관리하는 공간

회의 일정
정의
캘린더에 등록된 회의 일정

회의 참여자
정의
회의에 참석하는 사용자

회의록
정의
회의 내용을 기록한 데이터

AI 회의록
정의
회의 내용을 AI가 요약한 결과

회의 요약
정의
회의 핵심 내용을 요약한 정보

결정사항
정의
회의를 통해 최종 확정된 내용

액션아이템 (Action Item)
정의
회의 결과 도출된 실행 업무
혼동 주의
	•	업무항목이 아니다.
	•	업무요청 생성의 기반 데이터이다.
	•	액션아이템 상태는 회의록 내부 상태이며 업무항목 상태와 별개이다.
	•	업무요청으로 전환해도 업무항목이 자동 생성되지 않는다.

연차
Leave Management
정의
연차 및 휴가를 관리하는 화면

연차
정의
하루 전체 휴가

오전 반차
정의
오전 근무 면제

오후 반차
정의
오후 근무 면제

시간차
정의
시간 단위 휴가
혼동 주의
	•	MVP 범위에는 포함하지 않는다.
	•	추후 확장 후보이다.
	•	MVP 연차 유형은 종일 연차, 오전 반차, 오후 반차만 사용한다.

결재권자
정의
연차 승인 권한을 가진 사용자

승인대기
정의
아직 승인되지 않은 연차 상태

승인
정의
결재권자가 승인한 상태

반려
정의
결재권자가 승인하지 않은 상태

총 연차
정의
사용자에게 부여된 전체 연차

사용 연차
정의
이미 사용한 연차

잔여 연차
정의
사용 가능한 남은 연차

알림
알림 (Notification)
정의
시스템이 사용자에게 전달하는 이벤트 정보
예시
	•	업무 요청
	•	업무 수정
	•	업무항목 참여자 변경
	•	연차 승인
	•	연차 반려

읽지 않음
정의
사용자가 아직 확인하지 않은 알림 상태

읽음
정의
사용자가 확인한 알림 상태

Team Status
정의
팀 업무 운영 및 배정 현황을 관리하는 화면
혼동 주의
	•	단순 조회 화면이 아니다.
	•	팀장과 대표는 업무 운영 기능을 수행할 수 있다.

프로세스 관리
Process Management
정의
Manager가 반복적으로 사용하는 공통 업무 프로세스를 템플릿으로 등록하고 관리하는 화면
혼동 주의
	•	Manager 전용 기능이다.
	•	업무카테고리와 다른 개념이다.
	•	프로세스 템플릿은 업무요청 생성 시 자동 매칭되어 단계별 배정에 활용된다.

프로세스 템플릿 (Process Template)
정의
반복적으로 사용되는 업무 흐름을 단계별로 정의한 템플릿
예시
	•	배너관리 프로세스 → [기획 → 디자인 → 개발 → QA]
	•	영상 제작 프로세스 → [기획 → 촬영 → 편집 → 검수]
혼동 주의
	•	업무항목이 아니다.
	•	업무요청 생성 시 첨부되는 참조 데이터이다.
	•	프로세스 템플릿 자체는 업무를 생성하지 않는다.

프로세스 단계 (Process Step)
정의
프로세스 템플릿을 구성하는 순서가 있는 업무 단위
예시
	•	1단계: 기획
	•	2단계: 디자인
	•	3단계: 개발
혼동 주의
	•	작업세션이 아니다.
	•	업무항목이 아니다.
	•	단계별로 담당 팀과 담당자를 배정할 수 있다.

키워드 매칭 (Keyword Matching)
정의
업무요청 생성 시 프로젝트명에 포함된 키워드를 기반으로 프로세스 템플릿을 자동으로 제안하는 기능
예시
	•	"하반기 배너작업" 입력 → "배너" 키워드 감지 → "배너관리 프로세스" 제안
혼동 주의
	•	자동 적용이 아니라 제안이다.
	•	Manager가 제안된 프로세스를 선택 또는 무시할 수 있다.

KPI
정의
핵심 성과 지표
사용자의 업무 현황을 수치화하여 보여주는 데이터
예시
	•	오늘 작업시간
	•	완료 작업세션 수
	•	남은 작업세션 수
	•	진행중 업무 수
	•	지연중 업무 수

My Page
정의
개인의 업무 데이터와 이력을 조회하는 공간
혼동 주의
	•	총 연차, 사용 연차, 잔여 연차 현황은 포함하지 않는다.
	•	작업 히스토리 캘린더의 연차 상태 표시는 가능하다.
	•	연차 신청, 승인, 이력, 총/사용/잔여 연차 현황은 Leave Management에서 관리한다.
