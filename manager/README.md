# WorkFlow Hub — Manager Home Prototype

정적 HTML/CSS/JS로 구현한 팀장(Manager) 역할 인터랙티브 프로토타입.

## 실행

```bash
cd vibe-projects
python3 -m http.server 8787
# http://localhost:8787/manager/ 접속
```

## 데이터 컨텍스트

- **디자인 에이전시 시나리오**: 외부 클라이언트 회사로부터 업무요청을 수신하는 디자인 에이전시 맥락
- **동적 날짜 시프트**: 앵커 날짜(`_ANCHOR = 2026-06-13`) 기준으로 실행 시점의 오늘 날짜에 맞게 모든 더미 데이터 날짜를 자동 조정
- **새로고침 시 자동 셋팅**: 이번 주 업무는 실제 이번 주로, 오늘 할 일은 오늘 날짜로 자동 설정

---

## 레이아웃

```
[다크 사이드바 220px] | [메인 콘텐츠]
                           ├─ 헤더 (주간 날짜 필터 / 검색·알림·프로필)
                           └─ 3컬럼 그리드 (equal width)
                               ├─ col1: 이번 주 업무
                               ├─ col2: 오늘 할 일
                               └─ col3: 오늘의 현황 + 업무 요청
```

## 디자인 시스템

- **폰트**: DM Sans (UI) / DM Mono (KPI 숫자)
- **사이드바**: `#0e0e16` 배경, 활성 탭 inset 좌측 보더
- **CSS 변수**: `--bg`, `--surface`, `--surface-muted`, `--line`, `--line-soft`, `--text`, `--muted`, `--soft`, `--blue`, `--blue-soft`, `--shadow`, `--shadow-lg`, `--radius`, `--radius-sm`
- **모달/오버레이**: `backdrop-filter: blur` 적용

## 공통 헤더

- **주간 필터(좌)**: `< 2026 06.08 ~ 06.14 이번 주 >` 형태, 화살표로 주 단위 이동
- **검색(우)**: 헤더 내 인라인 인풋박스 (포커스 시 확장)
- **알림 팝오버(우)**: 읽음/안읽음 표시, 전체 읽음 처리, 개별 클릭 읽음 처리
  - 알림 아이콘: 읽지 않은 알림 수 뱃지 (99개 초과 시 `99+`), 모두 읽으면 숨김
  - 알림 레벨 배지(긴급/중요/일반) 없음
  - 업무요청 알림: 제목 옆에 업무항목명 파란 인라인 칩 표시
- **프로필(우)**: 사용자 아바타

### 알림 발생 케이스

| 알림 제목 | 트리거 |
|---|---|
| 업무요청 도착 | 타 팀에서 업무요청 전송 시 |
| 업무요청 수락 | 담당자가 업무요청 수락 시 |
| 업무요청 거절 | 담당자가 업무요청 거절 시 |
| 업무항목 추가 | 회의록 액션아이템 → 내 업무 추가 시 |
| 연차 신청 완료 | 본인이 연차 신청 제출 시 |
| 연차 승인 | Owner가 본인 연차 승인 시 |
| 연차 반려 | Owner가 본인 연차 반려 시 |
| 회의 등록 완료 | 본인이 회의 등록 시 (참석자 수 포함) |

## Home 페이지

### col1: 이번 주 업무 (Weekly Tasks)

- 요일(월~일)별 카드로 그룹화 (7일)
- 오늘 카드는 연한 파란 배경, 나머지는 연한 그레이 배경
- 카드 내부: 요일명(bold) + 날짜(MM/DD) → 아이템 목록
- **날짜(MM/DD) 클릭** → 오늘 할 일 섹션이 해당 요일 기준으로 전환
- 각 아이템: 유형 아이콘 + 업무명 + 기간/반복 정보
  - 유형 아이콘: 핀(고정) · 빨간 원(긴급) · 그레이 원(일반)
- 카드 내 고정 → 긴급 → 일반 순 정렬, 마지막 고정업무 아래 구분선
- 지연 상태(end < today + 미완료 세션 존재)일 때만 `지연중` 배지 (고정업무 제외)
- **업무명 클릭** → 오늘 할 일 패널에 해당 업무의 세션 인라인 입력창 표시 (토글)
- **편집 버튼** (hover 시 노출) → 업무 상세/수정 슬라이드 패널 오픈
- **삭제 버튼** (hover 시 편집 버튼 옆 노출) → 삭제 확인 모달 → 업무항목 + 관련 세션 전부 삭제
- `+ 업무 추가` → 요일 탭 기반 드로어 (우측 슬라이드, 500px)

**업무 추가 드로어:**
- 헤더: "Weekly Tasks" + X 닫기 버튼
- 요일 탭: 월/화/수/목/금 — 균등 너비, 활성 탭 검정 배경/흰 글씨
- 탭별 업무 카드: 프로젝트명 / 시작일·종료일(2컬럼) / 업무유형 / 메모
- **고정 유형 선택 시**: 시작일·종료일(2컬럼) + 반복 요일 토글(월~금 버튼) 표시
- `+ 업무 추가` 점선 버튼으로 같은 날에 카드 추가 가능
- `저장하기` 전체폭 버튼으로 저장

**고정업무 반복 규칙:**
- `recurringDays` 배열로 반복 요일 지정 (1=월~5=금), 최소 1개 유지
- `end: null`이면 무기한 반복, 날짜 지정 시 종료일까지만 표시
- 기간 표시 대신 `매주 월·수·금` 형태로 표시

**업무항목 데이터 구분:**
- **업무요청 수락 항목** (`sourceRequestId` 존재): 클라이언트 프로젝트명, 외부 참여자 배정
- **직접 추가 항목** (`sourceRequestId` 없음): 내부 업무, 참여자는 본인(Jihye)만

### 업무 상세/수정 슬라이드 패널 (500px)

- 편집 버튼 클릭 → 우측 오버레이 슬라이드 패널 (배경 딤처리)
- X 버튼(오른쪽 상단) 또는 배경 클릭으로 닫기
- 드로어와 동일한 `dw-field` 폼 스타일로 통일
- **업무요청 수락 항목** (`sourceRequestId` 존재):
  - 상단에 `업무요청` 파란 배지 표시
  - 업무명·업무유형·시작일·마감일 수정 불가 (disabled + opacity:0.5)
  - 설명만 수정 가능
  - 참여자 목록 표시
- **직접 추가 항목** (`sourceRequestId` 없음):
  - `업무요청` 배지 미표시
  - 업무명·업무유형·시작일·마감일·설명 모두 수정 가능
  - 참여자 목록 미표시
- 작업세션 목록 표시 (선택 날짜 기준, 카테고리 배지 없음)
- 변경 사항 있을 때만 `저장하기` 버튼 활성화 → 저장 시 workItems 업데이트 + 패널 자동 닫힘

### col2: 오늘 할 일 (Daily Todo)

- **기본 날짜**: 오늘 (페이지 로드 시 자동 설정)
- 선택 날짜의 모든 세션을 **시작시간 오름차순** 플랫 목록으로 표시 (시간 미입력 세션은 맨 아래)
- **날짜 전환**: Weekly Tasks 날짜(MM/DD) 클릭 또는 날짜 피커로 변경
  - 오늘이면 "오늘 할 일", 다른 날이면 "MM/DD (요일) 할 일" 표시
- **세션 추가**: 업무명 클릭 → **최상단**에 인라인 입력창 표시 (업무명 레이블 포함)
  - 세부항목명 입력 후 Enter → 세션 추가 (목록 맨 아래 배치), 입력창 닫힘
  - 세션 생성 시 시간은 비어 있음 (작업 완료 후 별도 입력)
  - 카테고리 선택·표시 없음 (제거됨)
- **세션 제목 수정**: 더블클릭 → 인라인 input → Enter 저장 / Esc·blur 취소
- 체크박스로 완료 토글 (완료 = 파란색 체크, 제목 취소선)
- **시간 입력**: 숫자 4자리 → HH:MM 자동 포맷 (예: `0923` → `09:23`)
  - Tab: 시작→종료 시간 이동 (저장만, 재정렬 없음)
  - Enter: 저장 + 시간순 재정렬 / Esc: 취소
  - 오늘 세션: 현재 시각 이후 입력 불가
  - 겹치는 시간 범위 입력 불가
  - 오류 시 인풋 위 툴팁 표시 후 원래 값 복원
  - 과거 날짜 세션은 제한 없음
- 세션 삭제: hover 시 삭제 버튼 → 확인 모달 → 물리 삭제
- 진행바: 완료/전체 진척률 (그라디언트)

### col3 상단: 오늘의 현황 (KPI)

- 1x2 KPI 그리드, 고정 높이
- 작업시간(파랑) / 세션 완료율(초록) — 선택 날짜 기준 계산
- 오늘이면 "오늘 작업시간", 다른 날이면 "MM/DD 작업시간" 라벨

### col3 하단: 업무 요청

- 오늘의 현황 아래 나머지 공간 전부 차지, 내부 스크롤
- 수락 대기 → 상단, 수락됨·거절됨 → 하단 정렬
- 각 요청: 업무명 + 상세설명 + 요청자·시작일~마감일 + 수락/거절 버튼
- **수락** → 업무항목 추가 모달:
  - 업무항목명: 업무유형 배지(긴급/일반) + 제목 + 기간
  - 작업 단계: 요청자가 선택한 하위 프로세스 단계 칩 리스트 (나에게 배정된 단계만 표시)
  - 작업 날짜: 선택한 날짜의 오늘 할 일에 자동 추가
  - 수락 완료 시: 이번 주 업무에 업무항목 추가 + 선택된 모든 단계를 세션으로 생성 + `수락됨` 배지 표시
  - 작업 날짜가 업무 기간 밖이면 start/end 자동 보정
- **거절** → 사유 선택 모달 (고정 목록 + 기타 상세) → `거절됨` 배지 + 버튼 숨김
- **주간 자동 리셋**: 앱 로드 시 이전 주 월요일 이전에 확정된 요청 자동 제거

## 기타 탭 (구현 완료)

### Calendar — 월간/주간 캘린더
- 헤더: `[전체 | 나]` 필터 + `[월간 | 주간]` 토글 + `< >` 이동
- 월간: 비반복 업무 → 가로 바, 반복 업무 → 세부항목 dot 리스트
- 주간: 비반복 업무 → 칩, 반복 업무 → dot 리스트
- 업무 클릭 → 상세 패널 (참여자별 작업세션 그룹 + 아웃풋/리소스)

### Team Status — 팀원 업무 현황
- 팀원별 업무항목 카드 그리드 (아바타 + 이름 + 역할 + 업무 목록 + 완료율)
- 상단 KPI 4종: 미배정 업무 / 신규 요청 / 수락 대기 / 7일 내 마감
- 업무배정 현황: 신규요청·수락대기·미배정·배정완료 그룹별 목록
- `+ 업무요청 전송` 버튼 (Manager/Owner만)

### Process Management — 프로세스 템플릿 관리
- 프로세스 목록 렌더링 (생성/수정/삭제 CRUD는 미구현)

### Meeting Room — 녹음 + 회의록
- 녹음: 마이크 → 파형 + 타이머 → 정지 → 저장 모달 (참석자 검색, 액션아이템 추가)
- 회의 목록: 팀 필터 탭 + 검색 + 카드 목록 + hover 삭제
- 회의록 상세: AI 분석 / 스크립트 / 액션아이템 3탭
- 액션아이템 `추가` 버튼 → 이번 주 업무에 등록

### Leave Management — 연차 관리
- KPI 3종: 총 연차 / 사용 연차 / 잔여 연차
- 탭: 내 연차(대기 중) / 이력(처리 완료)
- 연차 신청 모달: 유형(종일/오전 반차/오후 반차) + 시작일/종료일 + 사유
- 취소: 승인 대기 + 시작일 미도래 조건
- Manager/Owner: 타 팀원 대기 건 승인·반려

### My Page — 직원 대시보드
- 3컬럼: 프로필+카테고리별 작업시간 | AI 업무 분석+참여 회의 | 캘린더+세션 목록
- AI 업무 패턴: 세션 기반 6가지 인사이트 (데이터 없는 항목 자동 숨김)
- 미니 캘린더: 세션 있는 날짜 파란 점, 연차 날짜 주황 점
