# WorkFlow Hub - MVP Backlog

## 목적

본 문서는 WorkFlow Hub MVP를 실제 개발 작업 단위로 정리한다.

정책 기준은 `context/`, `rules/`, `knowledge/` 문서를 따른다.

## Epic 1. 기본 조직/권한

### 목표

Owner, Manager, Member 권한 체계를 기반으로 모든 기능의 접근 권한을 제어한다.

### 포함 기능

- 회사, 팀, 사용자 기본 구조
- 사용자 권한: Owner, Manager, Member
- Owner의 계정 생성 및 권한 부여
- 권한 변경 즉시 적용
- 권한 변경 이력 저장

### 제외 기능

- 세부 직무/직책 체계
- 복수 팀 소속
- 커스텀 권한 그룹

### 완료 기준

- Owner만 계정을 생성하고 권한을 부여할 수 있다.
- Manager는 소속 팀 기준으로 데이터를 조회/관리한다.
- Member는 본인 데이터 입력과 본인 작업세션 수정만 가능하다.
- 권한 변경 이력이 저장된다.

### 관련 문서

- `rules/role-permission.md`
- `rules/policy.md`
- `knowledge/data-model.md`

## Epic 2. Home

### 목표

사용자가 오늘 업무 현황을 확인하고 업무항목과 작업세션을 관리할 수 있다.

### 포함 기능

- KPI 위젯
- 업무요청 위젯
- Weekly Tasks 영역
- Daily Todo 영역
- 팀원 업무 현황
- 연차 사용 중인 사용자의 업무 KPI 비활성화

### 제외 기능

- Business Dashboard
- 매출/수익성 지표
- 타이머 기능

### 완료 기준

- 업무요청 수, 오늘 작업시간, 완료 작업세션 수, 남은 작업세션 수가 표시된다.
- Weekly Tasks는 업무항목 관리 영역으로 동작한다.
- Daily Todo는 선택한 업무항목의 작업세션 관리 영역으로 동작한다.
- 팀원 업무 현황에서 연차/오전 반차/오후 반차 상태가 업무 수치와 혼동되지 않게 표시된다.
- 연차 사용 중인 본인은 업무 KPI가 비활성화된다.

### 관련 문서

- `knowledge/features/home-feature-spec.md`
- `knowledge/features/leave-management.md`
- `DESIGN.md`

## Epic 3. Weekly Tasks / Work Item

### 목표

주간 업무항목을 계획하고 관리한다.

### 포함 기능

- 업무항목 직접 생성
- 업무요청 기반 업무항목 생성
- 시작일~마감일 기준 자동 배치
- 요일 섹션 컬러 표시
- 고정업무 아래 구분선 표시
- 고정 > 긴급 > 일반 정렬
- 업무유형: 일반, 고정, 긴급
- 업무항목 수정
- 업무항목 삭제 권한 처리
- 참여자 자동 등록
- 지연중 계산
- Daily Todo 자동 숨김 조건 처리

### 제외 기능

- 업무항목 완료 상태
- 상위/하위 프로젝트 구조
- 별도 프로젝트 엔티티

### 완료 기준

- 업무항목 생성 시 업무명, 시작일, 마감일, 업무유형이 필수이다.
- 직접 생성 시 생성자가 참여자로 자동 등록된다.
- 업무요청 기반 생성 시 담당자가 참여자로 자동 등록된다.
- 현재 날짜가 업무 기간에 포함되면 카드가 강조 표시된다.
- 컬러 강조는 업무항목 카드가 아니라 해당 요일 섹션에 적용된다.
- 고정업무 아래에만 구분선을 표시한다.
- 업무항목은 요일 안에서 고정, 긴급, 일반 순서로 정렬된다.
- 마감일 경과와 미완료 작업세션 존재 조건을 만족하면 지연중 태그가 표시된다.
- 마감일 경과와 미완료 작업세션 없음 조건을 만족하면 Daily Todo에서 자동 숨김 처리된다.
- 업무항목에는 완료 상태가 없다.

### 관련 문서

- `knowledge/features/home-feature-spec.md`
- `context/glossary.md`
- `knowledge/data-model.md`

## Epic 4. Daily Todo / Work Session

### 목표

업무항목 안에서 실제 작업 기록 단위인 작업세션을 생성하고 관리한다.

### 포함 기능

- 작업세션 생성
- 업무카테고리 선택
- 세부업무명 입력
- 시간 더블클릭 입력
- 시작시간/종료시간 저장
- 소요시간 자동 계산
- 완료 체크
- 작업세션 수정
- 작업세션 소프트 삭제
- 작업세션 수정 이력 저장

### 제외 기능

- 타이머 기능
- 업무항목 완료 처리

### 완료 기준

- 작업세션 최초 생성 시 업무카테고리와 세부업무명만 입력한다.
- 시간은 더블클릭 후 수동 입력한다.
- 시작시간과 종료시간 기준으로 소요시간이 자동 계산된다.
- 작업세션만 완료 체크할 수 있다.
- Owner/Manager는 작업세션을 수정/삭제할 수 있다.
- Member는 본인 작성 작업세션만 수정/삭제할 수 있다.
- 삭제 시 물리 삭제하지 않고 `deleted_at`, `deleted_by`를 기록한다.
- 소프트 삭제된 작업세션은 일반 화면에서 숨기고 분석에서 기본 제외한다.

### 관련 문서

- `knowledge/features/home-feature-spec.md`
- `rules/policy.md`
- `knowledge/data-model.md`

## Epic 5. Task Request

### 목표

팀 간 업무요청을 생성, 배정, 수락, 거절할 수 있다.

### 포함 기능

- 업무요청 생성
- 전달 팀 선택
- Manager의 담당자 배정
- 미배정 업무 상태
- 수락 대기 상태
- 담당자 수락
- 담당자 거절
- 고정 거절 사유 선택
- 기타 거절 상세 사유 입력
- 업무요청 수정
- 업무요청 이력 저장
- 미응답 알림
- 수락된 업무요청의 Weekly Tasks 추가

### 제외 기능

- 거절 사유 관리 화면
- 업무요청 수락 시 업무항목 자동 생성

### 완료 기준

- Owner/Manager만 업무요청을 생성할 수 있다.
- 업무명, 상세내용, 시작일, 마감일, 전달 팀, 중요도는 필수이다.
- 업무요청은 전달 팀으로 수신된다.
- 담당자 배정 전에는 미배정 업무로 표시된다.
- 전달 팀의 Manager가 담당자를 배정한다.
- 담당자 배정 후 담당자만 수락/거절할 수 있다.
- 거절 시 고정 사유를 선택해야 한다.
- 기타 선택 시 상세 사유가 필수이다.
- 수락해도 업무항목은 자동 생성되지 않는다.
- 사용자가 원하는 날짜를 선택해 Weekly Tasks에 추가한다.
- 24시간 미응답 시 담당자에게 1차 알림을 발송한다.
- 48시간 미응답 시 담당자에게 2차 알림과 요청자 알림을 발송한다.
- 자동 거절은 없다.

### 관련 문서

- `knowledge/features/task-request.md`
- `rules/policy.md`
- `knowledge/data-model.md`

## Epic 6. Calendar

### 목표

업무항목과 회의 일정을 Team/My 필터로 조회한다.

### 포함 기능

- Team 필터
- My 필터
- 업무항목 일정 표시
- 회의 일정 표시
- 업무항목 상세 조회
- 회의 상세 이동
- 캘린더 검색
- 권한별 수정/삭제 처리

### 제외 기능

- 연차 표시
- 업무요청 표시
- 작업세션 표시
- 캘린더에서 업무 생성

### 완료 기준

- Calendar에는 업무항목과 회의 일정만 표시된다.
- Team 필터는 팀 전체 일정을 보여준다.
- My 필터는 본인이 참여 중인 일정만 보여준다.
- Owner/Manager는 Team 필터에서 수정/삭제 가능하다.
- Member는 Team 필터에서 조회만 가능하다.
- My 필터는 조회 기능이며 수정 권한과 무관하다.
- 연차, 업무요청, 작업세션은 캘린더에 표시되지 않는다.

### 관련 문서

- `knowledge/features/calendar.md`
- `rules/role-permission.md`

## Epic 7. Meeting Room

### 목표

회의를 생성하고 AI 회의록과 액션아이템을 관리하며 업무요청으로 전환할 수 있다.

### 포함 기능

- 회의 생성
- 회의 녹음 상태 관리
- AI 회의록 저장
- 회의 요약
- 핵심 논의사항
- 스크립트
- 액션아이템 생성/수정
- 액션아이템 상태 관리
- 액션아이템 업무요청 전환
- 회의록 목록/상세/검색
- 회의 참여자 관리

### 제외 기능

- 액션아이템을 업무항목으로 직접 전환
- 회의 액션아이템 상태를 업무항목 상태로 동기화

### 완료 기준

- Owner/Manager/Member 모두 회의를 생성할 수 있다.
- 회의 녹음 상태는 대기, 녹음중, 완료로 관리한다.
- AI 분석 결과는 수정 가능하다.
- 액션아이템 상태는 진행 예정, 진행 중, 완료로 관리한다.
- 액션아이템 상태는 회의록 내부 상태이며 업무항목 상태와 별개이다.
- 액션아이템을 업무요청으로 전환해도 업무항목은 자동 생성되지 않는다.
- 업무요청 수락 후 사용자가 Weekly Tasks에 추가해야 업무항목이 된다.
- 회의 생성자, Owner, Manager가 회의 참여자를 관리할 수 있다.

### 관련 문서

- `knowledge/features/meeting-room.md`
- `knowledge/data-model.md`

## Epic 8. Leave Management

### 목표

연차 신청, 승인, 반려, 취소, 이력, 현황을 관리한다.

### 포함 기능

- 연차 신청
- 연차 승인
- 연차 반려
- 연차 취소
- 연차 현황 조회
- 연차 이력 조회
- 연차 사유 조회 권한 처리
- 홈 화면 연차 상태 반영

### 제외 기능

- 시간차
- Calendar 연차 표시
- My Page 총/사용/잔여 연차 현황

### 완료 기준

- MVP 연차 유형은 종일 연차, 오전 반차, 오후 반차만 제공한다.
- 신청 사유는 필수이다.
- Member 신청은 Manager가 승인한다.
- Manager 신청은 Owner가 승인한다.
- 반려 사유는 필수이다.
- 연차 시작 전까지만 취소할 수 있다.
- 캘린더에는 연차를 표시하지 않는다.
- My Page에는 총/사용/잔여 연차 현황을 표시하지 않는다.
- My Page 작업 히스토리 캘린더에는 연차 상태 표시만 허용한다.

### 관련 문서

- `knowledge/features/leave-management.md`
- `knowledge/features/my-page.md`
- `rules/role-permission.md`

## Epic 9. My Page

### 목표

개인의 업무 기록과 생산성 데이터를 조회하고 분석한다.

### 포함 기능

- 개인 작업 히스토리
- 월간 캘린더 기반 작업 이력
- 연차 상태 표시
- 개인 성과 요약
- 생산성 분석
- AI 업무 분석
- 계정 정보

### 제외 기능

- 총 연차
- 사용 연차
- 잔여 연차
- 연차 신청/승인/이력 관리

### 완료 기준

- 작업 히스토리에서 날짜별 작업세션을 조회할 수 있다.
- 작업 히스토리 캘린더에서 연차 상태 표시는 가능하다.
- 총/사용/잔여 연차 현황은 표시하지 않는다.
- 월별 작업시간, 완료 작업 추이, 카테고리별 작업 비율을 표시한다.
- AI 업무 분석은 많이 수행한 업무, 많이 소요된 업무, 생산성 높은/낮은 요일, 집중 시간대를 제공한다.
- 계정 정보는 이름, 소속 팀, 권한, 입사일을 표시한다.

### 관련 문서

- `knowledge/features/my-page.md`
- `DESIGN.md`

## Epic 10. Category Management

### 목표

작업세션 분석용 업무카테고리를 관리한다.

### 포함 기능

- 업무카테고리 목록 조회
- 업무카테고리 생성 요청
- 업무카테고리 수정 요청
- 업무카테고리 비활성화 요청
- 예약 변경 저장
- 매월 1일 적용
- 기존 작업세션 데이터 보존

### 제외 기능

- 업무카테고리 삭제
- 즉시 반영

### 완료 기준

- Owner만 업무카테고리를 생성/수정/비활성화할 수 있다.
- Manager/Member는 조회만 가능하다.
- 생성/수정/비활성화 요청은 즉시 반영하지 않고 예약 변경으로 저장한다.
- 예약 변경에는 `effective_from` 값이 저장된다.
- `effective_from`은 다음 월 1일로 설정된다.
- 매월 1일 예약 변경을 실제 카테고리 상태에 반영한다.
- 기존 작업세션의 카테고리 정보는 변경하지 않는다.
- 카테고리는 삭제할 수 없다.

### 관련 문서

- `knowledge/features/category-management.md`
- `knowledge/data-model.md`

## Epic 11. Notification

### 목표

주요 이벤트를 사용자에게 알리고 읽음 상태와 보관 정책을 관리한다.

### 포함 기능

- 알림 목록 조회
- 읽음/읽지 않음 상태
- 알림 클릭 시 읽음 처리
- 수동 읽음 상태 변경
- 전체 읽음 처리
- 업무요청 알림
- 업무요청 미응답 알림
- 업무항목 참여자 변경 알림
- 연차 알림
- 회의 알림
- 알림 우선순위
- 알림 보관 정책

### 제외 기능

- 사용자 알림 삭제
- 사용자별 알림 보관 기간 설정

### 완료 기준

- 사용자는 수신한 알림을 조회할 수 있다.
- 알림 클릭 시 읽음 처리된다.
- 전체 읽음 처리가 가능하다.
- 사용자는 알림을 삭제할 수 없다.
- 일반 알림 목록에는 최근 90일 알림만 표시된다.
- DB에는 1년간 보관한다.
- 긴급 업무요청, 마감일 변경, 긴급 업무 지정은 긴급 우선순위로 표시된다.

## Epic 12. Audit / History

### 목표

권한 변경과 주요 데이터 변경 이력을 저장한다.

### 포함 기능

- 권한 변경 이력
- 업무요청 변경 이력
- 작업세션 수정 이력
- 업무카테고리 예약 변경 이력
- 작업세션 소프트 삭제 기록

### 제외 기능

- 일반 사용자용 전체 감사 로그 조회 화면

### 완료 기준

- 권한 변경 이력이 저장된다.
- 업무요청 생성, 담당자 배정, 수정, 수락, 거절 이력이 저장된다.
- 작업세션 생성, 수정, 완료 변경, 삭제 이력이 저장된다.
- 작업세션 삭제 시 `deleted_at`, `deleted_by`가 저장된다.
- 사용자 화면에 모든 수정 이력을 노출하지 않는다.

### 관련 문서

- `rules/policy.md`
- `knowledge/data-model.md`
